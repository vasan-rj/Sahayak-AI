The problem

In India, filling a form assumes you can read and write. Hundreds of millions can't — not the form's language, sometimes not any script.

A pension, a bank account, a ration card depends on trusting a stranger to fill it for you. The help today is a harried clerk, a paid agent, or a relative on the phone. None of them can see your documents and confirm each value in your own language.

How it works

One conversation. The form fills itself.

Accuracy-critical values — name spelling, date of birth — are read off a printed official document, the easiest vision task, not the hardest. Everything else is captured by voice. Each value shows on screen and is confirmed aloud before the next field.

Why it can't be a chatbox

It depends on live sight and voice.

All three Live API capabilities named in Problem Statement 1 are load-bearing here — remove any one and the tool collapses.

Sees

Continuous vision

The camera reads printed documents in real time — the core of the capture loop, not a one-shot upload.

Hears

Barge-in & correction

The user can interrupt mid-confirmation to fix a value; the agent stops and re-captures. A two-way conversation, not a form.

Feels

Reads vocal tone

If the user sounds frustrated or confused, the agent slows down and reassures — before they give up.

Translates

Live Translate

Switch Hindi ↔ Tamil mid-sentence and the agent follows — no toggle, no reset. English captions run throughout for observers.

The anti-bar is cleared. Typed into a chatbox, Sahayak doesn't exist — it teaches nothing and completes a real official task by seeing documents and hearing speech in real time.

The trust model

Perception is probabilistic. The record is deterministic.

  • Confirm before advanceNo field is saved without the user's spoken approval — that confirmation is the whole trust model.
  • Read documents, don't guessPrinted official cards are far more reliable than handwriting. The agent never invents a value it can't see or hear.
  • Append-only capture logOne entry per confirmed value, with its source — document or voice — hash-chained and tamper-evident. Every value traces to how it was captured.
Honest boundaries

It doesn't replace the human channel. For a fully illiterate user there's still a submit / thumbprint step. What Sahayak fixes is the capture: accurate, in-language, and confirmed by the applicant instead of taken on faith from a middleman.

Session content is ephemeral; the capture log stays local. Production adds on-device redaction of ID numbers before frames leave the phone.

One baked-in template today. Generalizing — parsing any uploaded form into a field-map — is the near-term step, pitched as roadmap, not overclaimed as built.

Run it

One laptop, one phone, one printed form.

Three tiers — browser (React) ↔ FastAPI proxy ↔ Gemini Live API. The proxy keeps the key server-side and owns the deterministic record. No database, no queue, no GPU.
